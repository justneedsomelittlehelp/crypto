# Experiments (`experiments/`)

All evaluation results, model history docs, training run artifacts, and experimental result JSONs.

## Living documents (read these for context)

| File | Purpose |
|------|---------|
| **`MODEL_HISTORY.md`** | ⭐ Living doc: every architecture decision from RNN → current best, with reasoning. Read this first. |
| **`STRATEGY.md`** | User's manual trading strategy (what the model is built around) |
| **`RUN_INDEX.md`** | Maps each `runs/run_*` folder to the eval that produced it |

## Subdirectories

| Directory | Purpose |
|-----------|---------|
| `archive/` | Superseded result JSONs (15 files). See `archive/README.md`. |
| `runs/` | Per-fold training checkpoints (~456 directories). Generated by `train_model()` in `src/models/trainer.py`. |

## How results flow

```
1. Run eval script → writes JSON to experiments/ root
2. Eval script writes per-fold checkpoints to experiments/runs/run_<timestamp>/
3. Document the run in EVAL_*.md and RUN_INDEX.md
4. Update MODEL_HISTORY.md with the architecture decision
5. When superseded, mv the JSON to experiments/archive/
```

## Conventions

- Active result files at root level (so eval scripts that hardcode paths still work)
- Old/superseded results in `archive/` (organizational only, no code references)
- Markdown docs are append-only (don't delete history, mark sections as superseded)
