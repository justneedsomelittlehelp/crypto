# Data (`data/`)

Raw and processed market data. Most files are gitignored — only small reference datasets (e.g., funding rate CSVs, FGI index) are committed.

## Files (committed)

| File | Purpose |
|------|---------|
| `fear_greed_index.csv` | Daily Fear & Greed Index from 2018-02 to present. Used for FGI-based regime detection in label computation. |
| `funding_rate_binance_2020_2024.csv` | BTC perpetual funding rate from Binance, Sep 2019 → Dec 2023. Source: GitHub `historical-funding-rates-fetcher`. |
| `funding_rate_gate_2024_present.csv` | BTC perpetual funding rate from Gate.io, Jan 2024 → present. Fetched via Gate.io public API. |
| `funding_rate_merged.csv` | Merged Binance + Gate.io funding rate. Generated by `src/data/funding_rate.py`. |

## Subdirectories

| Directory | Purpose |
|-----------|---------|
| `raw/` | Per-exchange raw OHLCV CSVs after scraping (gitignored). |

## Files at project root (NOT in this directory)

The OHLCV+VP feature CSVs live at the project root, not in `data/`. They're written by `src/data/__main__.py scrape`:
- `BTC_1h_RELVP.csv` — 1h timeframe (~85k rows)
- `BTC_15m_RELVP.csv` — 15min timeframe (~340k rows)
- `BTC_*_RELVP_metadata.json` — provenance metadata

Why at root? Historical convention. The eval scripts hardcode the path. If we wanted to move them into `data/`, we'd need to update many scripts.

## Gitignore

- `data/raw/` is gitignored (per-exchange CSVs are large and easily regenerated)
- `BTC_*_RELVP*` at project root is also gitignored
- The funding rate CSVs are force-added because they're small (~100KB total) and needed on Colab
